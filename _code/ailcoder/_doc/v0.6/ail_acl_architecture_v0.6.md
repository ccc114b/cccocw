# AIL 與 ACL 系統架構說明 v0.6 (規劃中)

## 概述

v0.6 版本將新增 **AICommander**，讓系統可以**自動回答執行者的問題**，無需人類介入。

## 新增功能

### AICommander

AICommander 了解整體架構（AIL/ACL），能夠根據人類的原始目標和系統能力做出合理回覆。

## 系統架構 (v0.6 - 規劃)

## 角色分工 (v0.6)

| 角色 | 職責 | 檔案系統權限 |
|------|------|-------------|
| **ailcoder** | CLI 工具，讀取 .ail 檔案 | - |
| **AICommander** | 規劃、回答問題 | ❌ 無 |
| **AIExecutor** | 執行任務，產生 ACL 指令 | ❌ 無 |
| **ACLHandler** | 實際檔案操作 | ✅ 有 |

## AICommander 初始化

AICommander 在初始化時會載入完整的系統知識：

```python
class AICommander:
    def __init__(self):
        # 1. 載入 AIL 框架知識
        self.ail_knowledge = """
## AIL (AI Language) 框架

AIL 是一個讓人類以目標導向指揮 AI 完成任務的 Python 框架。

### 核心模組：
- **Goal**: 目標追蹤
- **Agent**: AI 代理程式
- **Memory**: 長期記憶
- **Explore**: 平行探索
- **Think**: 推理思考
"""
        
        # 2. 載入 ACL 指令集知識
        self.acl_knowledge = """
## ACL (AI Communication Language)

ACL 是 AI 與系統之間的結構化溝通格式。

### 指令集：
- **<write file="...">**: 寫入檔案
- **<shell>**: 執行命令
- **<read file="...">**: 讀取檔案
- **<ask>**: 詢問指揮官
- **<answer>**: 回覆執行者
- **<result>**: 回報結果

### 重要規則：
1. 執行者沒有檔案系統權限
2. 所有檔案操作必須透過 ACL
3. 如有不清楚，發 <ask> 詢問
"""
        
        # 3. Ollama 執行者
        self.executor = OllamaAgent("ai-commander")
    
    def set_project(self, task: str):
        """設定專案目標"""
        self.project_context = task
    
    async def answer(self, question: str) -> str:
        """回答執行者的問題"""
        prompt = f"""
## 專案目標
{self.project_context}

## AIL 框架知識
{self.ail_knowledge}

## ACL 指令集
{self.acl_knowledge}

## 執行者詢問
{question}

請根據專案目標和 AIL/ACL 架構，直接回答執行者的問題。
"""
        return await self.executor(prompt)
```
