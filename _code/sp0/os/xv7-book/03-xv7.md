# 3. xv7 的擴充部分

## 3.1 什麼是 xv7？

xv7 是基於 xv6-riscv-net 的進一步擴充版本，新增了完整的 HTTP 伺服器和客戶端功能。這個版本讓 xv6 不僅能處理基礎的 UDP/TCP 通訊，還能作為 HTTP 伺服器提供網頁服務，或作為 HTTP 客戶端存取網路上的資源。

## 3.2 新增的功能

### 3.2.1 HTTP 伺服器 (httpd.c)

```c
// 簡化的 HTTP 伺服器
int main(int argc, char *argv[])
{
    int soc = socket(PF_INET, SOCK_STREAM, IPPROTO_TCP);
    // 監聽 port 8080
    listen(soc, 5);
    
    while (1) {
        int acc = accept(soc, ...);
        recv(acc, buf, sizeof(buf));
        
        // 處理 GET 請求
        if (buf[0] == 'G' && buf[1] == 'E' && buf[2] == 'T') {
            send(acc, HTTP_RESPONSE, ...);
        }
        close(acc);
    }
}
```

功能包括：
- TCP 連線監聽 (預設 port 8080)
- HTTP GET 請求解析
- HTTP 回應封裝 (status line、headers、body)
- 每個連線處理完後關閉

### 3.2.2 HTTP 客戶端 (curl.c)

```c
// 簡化的 HTTP 客戶端
int main(int argc, char *argv[])
{
    int soc = socket(PF_INET, SOCK_STREAM, IPPROTO_TCP);
    
    // 連線到伺服器
    connect(soc, (struct sockaddr *)&server, sizeof(server));
    
    // 發送 HTTP 請求
    char request[] = "GET / HTTP/1.1\r\nHost: ...\r\n\r\n";
    send(soc, request, strlen(request));
    
    // 接收回應
    int n = recv(soc, buf, sizeof(buf) - 1);
    printf("%s\n", buf);
}
```

功能包括：
- TCP 客戶端連線
- HTTP GET 請求構建
- 伺服器回應接收與顯示

### 3.2.3 Telnet 伺服器 (telnetd.c)

```c
// 簡化的 Telnet 伺服器
int main(int argc, char *argv[])
{
    int soc = socket(PF_INET, SOCK_STREAM, IPPROTO_TCP);
    
    // 監聽 port 23
    bind(soc, ...);
    listen(soc, 5);
    
    while (1) {
        int acc = accept(soc, ...);
        
        // 發送歡迎訊息
        send(acc, "Welcome!...\r\n");
        
        // 接收客戶端輸入
        while (recv(acc, buf, sizeof(buf)) > 0) {
            // 處理命令輸入
        }
        close(acc);
    }
}
```

功能包括：
- TCP 連線監聽 (預設 port 23)
- 歡迎訊息顯示
- 簡單的命令處理
- 客戶端連線管理

## 3.4 編譯方式

修改 Makefile 的 UPROGS 區段：

```makefile
UPROGS=\
    ...
    $U/_httpd\
    $U/_curl\
    $U/_telnetd\
    ...
```

編譯指令：
```bash
make TOOLPREFIX=riscv64-linux-gnu-
```

## 3.5 使用方式

### 啟動 HTTP 伺服器
```bash
# 在 xv6 中執行
ifconfig net0 192.0.2.2/24
httpd
```

### 使用 HTTP 客戶端
```bash
# 在 xv6 中執行
curl 192.0.2.2 8080
```

### 從主機測試
```bash
# 在主機端執行
curl http://192.0.2.2:8080
```

### 啟動 Telnet 伺服器
```bash
# 在 xv6 中執行 (預設 port 23)
ifconfig net0 192.0.2.2/24
telnetd

# 指定其他 port
telnetd 8023
```

### 從主機連線 Telnet
```bash
# 在主機端執行
telnet 192.0.2.2
```

## 3.6 技術要點

### 3.6.1 xv6 限制的處理

xv6 環境有諸多限制，需要特別處理：

1. **const 限定符問題**
   ```c
   // 錯誤：const char* 無法傳給 send()
   static const char *RESPONSE = "HTTP/1.1 200 OK...";
   send(soc, RESPONSE, ...);
   
   // 正確：使用 char[]
   static char RESPONSE[] = "HTTP/1.1 200 OK...";
   send(soc, RESPONSE, ...);
   ```

2. **缺少標準函式庫**
   - 沒有 strncmp、strcpy、strcat
   - 需手動實現字串操作
   - 使用指標逐一處理字元

3. **HTTP 請求解析**
   ```c
   // 避免使用 strncmp
   if (buf[0] == 'G' && buf[1] == 'E' && buf[2] == 'T' && buf[3] == ' ')
   ```

### 3.6.2 HTTP 協定實現

HTTP 回應格式：
```
HTTP/1.1 200 OK\r\n
Content-Type: text/html\r\n
Content-Length: 13\r\n
\r\n
<h1>Hello!</h1>
```

## 3.8 擴充方向

xv7 可以繼續擴充的方向：

1. **更完整的 HTTP 1.1 支援**
   - POST 請求處理
   - Cookie/Session
   - Keep-alive

2. **檔案傳輸**
   - HTTP 檔案下載
   - 靜態檔案服務

3. **安全傳輸**
   - HTTPS (需要 TLS 函式庫)

4. **效能優化**
   - 多執行緒處理
   - 連線池
