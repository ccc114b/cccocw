# 改進計劃：實現自我編譯 (Bootstrap)

## 專案現狀
- c0c: C 編譯器 (C → LLVM IR)
- ll0c: LLVM IR → RISC-V 目的檔
- rv0vm: RISC-V 虛擬機
- 支援: 算術/位元/邏輯運算、比較、条件判斷、迴圈、函式、指標、型別轉換
- 缺少: 結構體、陣列、標準庫、預處理器

---

## 第一階段：完善 C 編譯器 (c0c)

### 1.1 結構體/聯合體支援
- struct/union 類型定義
- 成員存取 (. 和 ->)
- 位元欄位

### 1.2 陣列支援
- 一維和多維陣列
- 陣列指標
- 動態配置

### 1.3 指標運算
- 完整指標算術
- void* 指標
- 函式指標

### 1.4 標準庫
- printf, fprintf, sprintf
- malloc, free, realloc
- strlen, strcpy, strcmp
- assert, exit

### 1.5 預處理器
- #define (巨集)
- #include
- #ifdef, #ifndef, #else, #endif
- #pragma

---

## 第二階段：完善 Runtime

### 2.1 浮點數運算
- 完整 FPU 指令支援
- float/double 類型
- 浮點數標準庫函数

### 2.2 除法/餘數
- 軟體實現或硬體模擬
- 負數除法規則

### 2.3 字串處理
- 字串操作函式
- 字面常數優化

---

## 第三階段：Bootstrap

### 3.1 將 c0c 用自身編譯
```
c0c c0c.c -o c0c.ll  # 用現有編譯器編譯自己
ll0c c0c.ll -o c0c_new.o  # 轉為 RISC-V
```

### 3.2 交叉編譯
- 在 x86 上編譯出 RISC-V 版 c0c

### 3.3 驗證自我編譯
- 用新編譯器再次編譯自己
- 比對輸出確認正確性

---

## 第四階段：自我改進

### 4.1 優化改進
- 使用新編譯器編譯自己
- 逐階段完善功能

### 4.2 迴歸測試
- 確保所有原有功能正常
- 驗證新功能正確性
