Google 工程師釋出的讓開發更有系統的 Agent Skill
AI 寫程式雖然很快，但它常常會走捷徑——跳過規格說明、不寫測試、忽略安全審查，只追求「完成」，而不追求「正確」。這套 Skills 的目的，就是為了解決這個問題。
覆蓋完整開發生命週期：
→ Define：先撰寫規格，再開始寫程式
→ Plan：拆解為小粒度、可驗證的任務
→ Build：採用增量實作、上下文工程、乾淨的 API 設計
→ Verify：導入 TDD、瀏覽器測試與系統化除錯
→ Review：進行程式碼品質檢查、安全強化與效能優化
→ Ship：整合 Git 工作流程、CI/CD、架構決策紀錄與上線清單
7 個斜線命令可直接使用：
/spec → /plan → /build → /test → /review → /code-simplify → /ship
內建 Google 工程文化的核心原則：
Shift Left（問題前移）
Chesterton's Fence（先理解再修改）
Hyrum's Law（所有可觀察的行為都會被依賴）
支援 Claude Code、Cursor、Antigravity，以及任何支援 Markdown 的 Agent。
