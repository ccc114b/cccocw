
# ccc: goose 和 opencode 兩者，定位有何不同嗎？

https://gemini.google.com/app/9b9bb7f9e4af6dc4

簡單來說，**Goose** 和 **OpenCode** 雖然都是開源的 AI 程式開發代理（AI Coding Agent），但在**底層設計理念**與**使用者互動方式**上有明顯的差異。

以下是兩者的定位對照：

### 1. Goose：強調「模組化與多代理協作」的開發者框架
Goose（由 Block/Block Inc. 開發）更像是一個**靈活的底層架構**。

* **核心定位**：它被設計為一個可以透過指令進行擴展的「代理（Agent）」。它不綁定特定的 AI 模型，通常與 **Ollama** 結合使用，讓你可以在本地運行開源模型（如 Qwen2.5-Coder）。
* **互動方式**：主要透過 **MCP (Model Context Protocol)** 與各種工具連接。它強調的是「可擴展性」，開發者可以透過定義特定的指令（Capabilities），讓 Goose 去操作電腦、讀寫檔案或與 API 互動。
* **適合對象**：喜歡自定義工作流、想在本地端（Local-first）建立強大開發代理系統的進階工程師。

### 2. OpenCode：強調「全通路與易用性」的開源 AI 助理
OpenCode 則更像是一個**產品級的開源替代方案**（對標 Claude Code 或 Cursor）。

* **核心定位**：它是一個即插即用的 AI 編碼代理，目標是提供一個更直覺、涵蓋更多前端界面的工具（包含 TUI 終端界面、桌面 App、IDE 擴充外掛）。
* **核心功能**：具備獨特的「計畫模式（Plan Mode）」，讓 AI 在動手改程式碼前先與你溝通邏輯。它也支援多模態（拖放截圖到終端機進行視覺分析），這在目前的命令行工具中較為少見。
* **適合對象**：想要一個功能完整、有良好 UI/UX 體驗，且能快速安裝並投入實際專案開發的工程師。

---

### 定位對比表

| 特性 | Goose | OpenCode |
| :--- | :--- | :--- |
| **主要定位** | 模組化代理框架 (Agent Framework) | 多功能開發代理工具 (Coding Tool) |
| **底層協議** | 深度整合 MCP (可擴充性強) | 強調多模態支援 (圖片/截圖) |
| **運作模式** | 像是一個「後端服務」，可串接多種工具 | 像是一個「前端產品」，提供 TUI 與桌面 App |
| **最大亮點** | 可自由定義 AI 的「能力（Toolkit）」 | 提供「計畫模式」，溝通邏輯後再構建 |
| **部署門檻** | 略高，需配置底層伺服器（如 Ollama） | 低，提供多種安裝腳本與 Zen 精選模型 |

**一句話總結：**
如果你想要**深度定制**一個屬於自己的 AI 開發機器人，選 **Goose**；如果你想要一個**開源且功能強大**、能直接取代訂閱制 AI 編輯器的工具，選 **OpenCode**。
