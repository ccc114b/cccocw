# LLM Wiki 操作手册

## 核心原则

1. **LLM 负责维护**: 所有 Wiki 页面由 LLM 编写和更新
2. **人类负责方向**: 人类提供来源、提出问题、引导分析
3. **持续累积**: 知识随时间累积，不是每次重新发现

## 操作流程

### 摄取 (Ingest)

```
1. 将新来源放入 source/ 目录
2. 告诉 LLM 处理该来源
3. LLM 会:
   - 读取来源
   - 与你讨论要点
   - 写入摘要页面
   - 更新索引
   - 更新相关概念/主题页面
   - 追加日志
```

### 查询 (Query)

```
1. 向 LLM 提问
2. LLM 搜索相关页面
3. 综合答案并引用来源
4. 好的答案可以存回 Wiki 作为新页面
```

### 整理 (Lint)

```
定期让 LLM 检查 Wiki 健康:
- 页面间的矛盾
- 过时被新来源取代的声明
- 孤立的页面（无入站链接）
- 缺少自己页面的重要概念
- 缺失的交叉引用
```

## 文件规范

### index.md

```markdown
# 索引

## 概念
- [編譯器](./概念/編譯器.md) - 將高階語言翻譯為機器碼

## 主題
- [RISC-V](./主題/RISC-V.md) - 精簡指令集架構
```

### log.md

```markdown
# 日志

## [2026-04-07] ingest | 系统程序概论
- 摄入了系统程序教材
```

## 来源管理

- 原始来源放在 `source/` 目录
- 来源是不可变的（LLM 只读不写）
- 每个来源对应一个摘要页
