# 有限狀態機 (Finite State Machine)

## 概述

有限狀態機（FSM）是計算理論中最簡單的計算模型，用於識別正則語言。

## 形式定義

```
FSM = (Q, Σ, δ, q₀, F)

Q  - 有限的狀態集合
Σ  - 輸入字母表
δ  - 轉移函數: Q × Σ → Q
q₀ - 起始狀態 (q₀ ∈ Q)
F  - 接受狀態集合 (F ⊆ Q)
```

## DFA vs NFA

| 特性 | DFA | NFA |
|------|-----|-----|
| 轉移 | 每個狀態只有一個 | 可能有多個 |
| ε-轉移 | 無 | 有 |
| 記憶體 | O(1) | 可能需指數空間 |

## Python 實作

```python
class FiniteStateMachine:
    def __init__(self, start, finals, actionMap):
        self.actionMap = actionMap
        self.start = start
        self.finals = finals
    
    def accept(self, s):
        state = self.start
        for c in s:
            state = self.actionMap.get(f'{state},{c}')
            if state is None: return False
        return state in self.finals
```

## 範例：aⁿbᵐ 語言

```python
fsm = FiniteStateMachine('s1', ['s2'], {
    's1,a': 's1',  # 讀取 a，保持 s1
    's1,b': 's2',  # 第一個 b
    's2,b': 's2',  # 繼續讀取 b
})
```

## 範例：偶數個 0

```python
fsm = FiniteStateMachine('s1', ['s1'], {
    's1,0': 's2',  # 讀到 0：奇數個 → s2
    's1,1': 's1',
    's2,0': 's1',  # 讀到 0：偶數個 → s1
    's2,1': 's2',
})
```

狀態意義：
- `s1`: 到目前為止讀到偶數個 0
- `s2`: 到目前為止讀到奇數個 0

## FSM 的限制

| 語言 | 原因 |
|------|------|
| aⁿbⁿ | 需要記憶計數器 |
| aⁿbⁿcⁿ | 需要兩個計數器 |
| 匹配的括號 | 需要棧來追蹤深度 |

## 與正則表達式的關係

```
正則表達式 (Regex)
       ↓ 等價
有限狀態機 (FSM)
       ↓ 等價
正則文法 (Regular Grammar)
```

## 實際應用

1. **詞彙分析器 (Lexer)**: Regex → NFA → DFA
2. **協定設計**: TCP 狀態機、HTTP 狀態碼
3. **硬體設計**: 數位電路、狀態機
4. **遊戲 AI**: 狀態機器人
