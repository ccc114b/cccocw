# 持續整合

持續整合 (CI) 將程式碼變更自動合併到共享分支並進行驗證。

## 基本流程

```
開發者 push → CI 伺服器 → 建構 → 測試 → 回饋
```

## GitHub Actions

```yaml
name: CI
on: [push, pull_request]
jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - name: Setup Node
        uses: actions/setup-node@v3
        with:
          node-version: '18'
      - name: Install
        run: npm install
      - name: Test
        run: npm test
      - name: Build
        run: npm run build
```

## 最佳實踐

- 小而頻繁的提交
- 保持快速的建構時間
- 修復失敗的建構優先
- 測試應接近開發環境
